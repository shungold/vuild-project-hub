Step 1: Structure Analysis

Read all channel messages + all thread replies. Analyze the channel composition and detect patterns.

Rules applied at this step Channel Name Rule (use Slack channel name as-is) / Scan Target Range (messages + all threads) / Team Extraction (users.info API, VUILD vs subcontractor)

5 Detection Axes → Patterns

AxisOptionsTriggers
Furniture countSingle / MultipleB Complex furniture composition rules
Area compositionSingle / Multi-floorB Area × Furniture mapping
Construction workNone / MixedC Furniture vs construction routing
WS eventsNone / PresentD WS event analysis
Year / RelatedSingle / Multi-yearE Separate projects + related_projects

Pattern A: Simple — No additional processing

A Few furniture, single area, no construction, no WS. Straight to Step 2.

Pattern B: Multi-Area

B Create Area × Furniture Map. Referenced by ALL subsequent steps (photo classification, specs, pins).

Area Mapping Rules

Explicitly map which furniture goes to which area/floor. User must confirm. Complex furniture splits by floor / type / block count.

Existing furniture detection: search furniture.json → confirm with user if same or new.

CLAUDE.md: Area Mapping / Complex Furniture / Existing Furniture Detection

Pattern C: Construction Mixed

C Route information: VUILD-made & movable → furniture / wall/floor/ceiling-fixed → project page.

Pattern D: WS Events

D Detect WS events, analyze participants/date/furniture. Distinguish WS photos vs production knowledge vs post-completion.

WS Photo Rules

Participant working = WS photo (max 5/event). Craftsman working = production knowledge. Don't judge by filename alone.

CLAUDE.md: WS Event Management

Pattern E: Multi-Year / Related

E Always separate projects. No year tabs. Link via related_projects with relation type (ws/year/phase/related).

User Confirmation Gate Present structure analysis report. Wait for user confirmation/corrections before proceeding.

2-6. Photo Selection (3 Phases)

Phase A: Text Signal Classification

Signal Priority

Message text (highest) > Thread parent (high) > Time cluster (high) > Nearby 5 messages (mid) > Poster pattern (mid) > Timestamp (mid)

Batch posts (3+ photos, no text): send ALL to Phase B. Judge each photo individually.

Phase B: Image Recognition

Target & Rules

Top 10 per group + undetermined + batch photos. Match against Step 1 furniture features.

B Area map: subject must be >50% area or center. Background = NG. Multi-furniture panorama = project only.

D WS photos: participant working = WS tab. Craftsman = production. Don't judge by filename.

ViewCapture hero check: 3D perspective = OK / CAM path, 2D drawing, nesting = NG

Step 3: Cross-Check

After distributing all data, open each furniture's "room" and verify holistically. Catch contamination, contradictions, and gaps that per-item classification misses.

4 Checks

Check 1: Contamination
Does every item in this room actually belong here?
Example: 6F paint info found in 7F kitchen's room → move to 6F
Check 2: Consistency
Do specs, knowledge, photos, and costs agree with each other?
Example: specs say "hinoki plywood" but knowledge says "lauan" → ask user
Check 3: Gaps
Is any furniture missing data that should exist?
Example: furniture has photos but zero knowledge/cost → check project room
Check 4: Project Overview
Is the project balanced? Do costs add up?
Example: furniture total ≠ project total → routing error possible
