## WEBの画面構成

### ポータルページ（index.html）
```
[案件リスト]  [全案件の金物]
```
- **案件リスト**: カードをクリック → 案件詳細へ
- **全案件の金物**: 全プロジェクトの金物を横断検索

### 案件詳細ページ（projects/xxx/index.html）
```
[アクソメ]  [図面]  [金物カタログ]  [BOM]  [ナレッジ]
```

#### アクソメビュー
- 什器のアクソメ図にピン（赤い●）が配置
- ピンクリック → 右サイドバーに金物詳細
- 黄色ピン = 未登録金物

#### 図面プレビュー
- 用紙サイズ: A1/A2/A3 切替
- 縮尺: 1:1〜1:50 切替
- ビューをドラッグで用紙上にレイアウト
- レイヤトグル: 構造/金物/設備/隠れ線/寸法
- PDF出力ボタン

#### 金物カタログ
- **Slackからの候補**（黄色ダッシュ枠）
  - 「採用する」→ 正式登録 + 編集モーダル
  - 「却下」→ 非表示
- **登録済み金物**（通常カード）
  - クリック → 編集モーダル（品名/品番/メーカー/価格/サムネ/履歴）
  - 重複品番の自動検知
  - AIサーチボタン（品番/品名で類似品検索）
- 検索 + カテゴリフィルタ

#### BOM（概算見積もり）
- カテゴリ: 家具/金物/シンク・ポンプ系/電源系/塗料/外注費/社内工数/管理費/輸送費/消耗品費
- 原価・数量は直接編集可能
- 単位プルダウン（枚/個/台/セット/式/人工/時間）
- 発注チェックボックス
- 製作予算 → 原価・粗利・粗利率をリアルタイム計算
- **JSONインポート**: AUTOESTIMATORのJSONをドラッグ&ドロップ
- **JSONエクスポート**: BOM全データをJSON出力
- CSVエクスポート

#### ナレッジ
- Slackのスタンプ付き投稿を自動収集
- フィルタ: 📝経過メモ / ⚠️トラブル / 💡ハウツー / 🚩仕様決定
- 検索（タイトル・内容・投稿者・チャンネル横断）
- 各カードに「Slackで見る」リンク

## Slackスタンプルール

### 金物用（🔩金物選定スレッド内）
| スタンプ | 意味 | WEB動作 |
|---------|------|---------|
| ✅ | 本採用 | カタログに正式登録 |
| 🔍 | 検討中 | 候補として黄色枠表示 |
| ❌ | 却下 | 候補から除外 |
| 💰 | 発注済み | BOMで発注チェックON |

### ナレッジ用（チャンネル内どこでも）
| スタンプ | 意味 |
|---------|------|
| :memo: | 経過メモ |
| :warning: | トラブル |
| :bulb: | ハウツー |
| :triangular_flag_on_post: | 仕様決定 |

## AUTOESTIMATOR連携

1. https://aiestimate-production.up.railway.app/ で見積もり生成
2. JSONをエクスポート
3. WEBのBOMタブでドラッグ&ドロップ
4. 材料費/加工費/人工が自動でBOMカテゴリに振り分け
5. 金物はhardware.jsonと照合 → 概算 vs 実価格の差分表示

## データの流れ

```
新チャンネルを発見
  ↓ 「初回スキャン {チャンネルID}」
Step 1: 構造分析 → ユーザー確認
Step 2: データ生成（ナレッジ/金物/写真/コスト/ドキュメント）
Step 3: クロスチェック → ユーザー確認
Step 4: ページ生成 → git push
Step 5: WEB確認
  ↓
以降は「Slackスキャンして」で差分更新
  ↓ 「GitHubにpushして」
公開サイトに反映（1-2分）
```
